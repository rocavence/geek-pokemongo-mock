這份指南在幹嘛

舊手機原廠停更後，裝自訂 ROM 補安全性更新、或 root 來除惡意預載，本身能提升安全；但解鎖 bootloader 會讓 Play Integrity 認證失敗，連帶銀行、Google 錢包、Netflix 等正常 App 拒絕啟動。這份就是把自己的裝置重新調到通過認證，好用回這些 App。

  • 只處理「通過認證 / 隱藏 root」這一層，目的是自用 App 能跑
  • 你自己的裝置、合法改機
  • 不含 GPS 假定位、不含繞過任何遊戲反作弊（那是作弊，不在這）

先搞懂：三種 verdict

SafetyNet 已退役，現在全看 Play Integrity API 的三級判定。先確認你的 App 到底要哪一級，再決定要做到多深 —— 多數銀行只要 BASIC。

  • MEETS_BASIC_INTEGRITY裝置大致正常。多數銀行 / 一般 App 只查這個。純軟體模組就能過。
  • MEETS_DEVICE_INTEGRITY通過 Google 認證的裝置。較嚴的金融 / 錢包要這個。
  • MEETS_STRONG_INTEGRITY硬體背書（TEE）。少數高安全 App，需有效 keybox。

2025/5 起的關鍵規則

DEVICE 在 Android 13+ 要求「bootloader 已上鎖」訊號。root 必須解鎖，所以 A13 想拿 DEVICE 得靠 TrickyStore 偽裝上鎖加上有效 keybox。A12 以下不受此限，DEVICE 好拿很多。

兩支目標機難度不同

verdict小米8 · A10小米11 Lite · A13
BASIC易 PIF 單獨搞定易 PIF 單獨搞定
DEVICE中 不受上鎖新規卡，PIF 指紋多半夠難 要 TrickyStore 偽裝上鎖 + 有效 keybox，會隨撤銷失效
STRONG難 要 TrickyStore + 有效 keybox難 要 TrickyStore + 有效 keybox

標準模組棧（2026 維護中版本）

  • Magisk 28+（含 Zygisk）— root 與模組框架。repo
  • Shamiko — 隱藏 root；DenyList 設好，但 Magisk 的「強制執行拒絕清單」要關掉讓 Shamiko 接手。repo
  • Play Integrity Fix（維護分支） — 過 BASIC，A10 多半也過 DEVICE。osm0sis/PlayIntegrityFork 或 KOWX712
  • TrickyStore + keybox.xml — 只有 A13 要 DEVICE、或要 STRONG 才加。repo

前置：已解鎖 bootloader、電腦裝好 ADB（Mac：brew install android-platform-tools）。

安裝順序（從 Magisk root 開始）

順序會影響成敗。原則：每裝一個模組就重啟一次，裝越少越穩，需要才往下加。第 1～5 步是所有人都要的底；第 6 步之後只有要 DEVICE（A13）或 STRONG 才裝。

  1. 刷入 Magisk root：用 Magisk 修補官方 ROM 的 boot.img，fastboot flash boot magisk_patched.img，開機後確認 Magisk 顯示「已安裝」。
  2. 隱藏 Magisk：Magisk → 設定 → 隱藏 Magisk 應用程式（給隨機名稱），等重新封裝完成。
  3. 開 Zygisk：Magisk → 設定 → Zygisk = 開 → 重啟。（用 KernelSU / APatch 而非 Magisk 才需改裝 ZygiskNext，與內建 Zygisk 二擇一。）
  4. Shamiko：Magisk → 設定 → 強制執行拒絕清單：關；配置拒絕清單勾 Google Play 服務、Play 商店、目標 App。再裝 Shamiko 模組 → 重啟。
  5. Play Integrity Fix（osm0sis 分支）：裝模組 → 重啟 → 測 BASIC。多數人到這就夠了（接路線 A）。
  6. TrickyStore：裝 → 重啟 → 放 keybox.xml 到 /data/adb/tricky_store/、設 target 含 com.google.android.gms。（A13 要 DEVICE / 任何機要 STRONG 才需要。）
  7. Tricky-Addon + KSUWebUI（選用）：Tricky-Addon 自動連網抓 keybox；若 Magisk 內 Action 按鈕不穩或消失，改用 KSUWebUI 的獨立設定頁操作 Addon。
  8. LSPosed Mod → Hide My Applist（選用）：裝 LSPosed Mod → 重啟 → 裝 HMA，對目標 App 隱藏 root 相關 App 清單。少數會掃 App 清單的 App 才需要。

為什麼是這個順序

先有 root 與隱藏（1～4）才談得上認證偽裝（5 以後）；DenyList 要在裝 Shamiko 前設好，Shamiko 才接得到。每步之間重啟，避免模組狀態沒寫定就疊下一個。

路線 A：只過 BASIC（先試這條）

很多人卡關只是因為一上來就上 keybox。先用最省的：

  1. Magisk 裝好，確認 Zygisk = 開，重啟。
  2. 裝 Play Integrity Fix 模組，重啟。
  3. Magisk → 設定 → 強制執行拒絕清單：關；裝 Shamiko，重啟。
  4. 配置拒絕清單勾選會檢測的對象：Google Play 服務、Play 商店、以及你要用的那個銀行 App。
  5. Magisk → 設定 → 隱藏 Magisk 應用程式（給隨機名稱）。

清一次 GMS 快取讓判定重跑：

adb shell pm clear com.google.android.gms
adb shell pm clear com.android.vending

測一下（見下方驗證）。過 BASIC 且你的 App 能用，就停在這，不用碰 keybox。

路線 B：要過 DEVICE

小米8（A10）

通常路線 A 的 PIF 指紋就能順帶過 DEVICE。若沒過，重生一份指紋：

# 重生 fingerprint（模組目錄名以實際為準）
adb shell su -c "cd /data/adb/modules/playintegrityfork && rm -f custom.pif.json"
# 再開 Magisk → 模組 → PlayIntegrityFork → 動作按鈕，重新產生

小米11 Lite（A13）

受「上鎖才給 DEVICE」限制，要加 TrickyStore 偽裝上鎖加 keybox：

  1. 裝 TrickyStore 模組，重啟。
  2. 放入一把有效（未被撤銷）的 keybox.xml 到 /data/adb/tricky_store/keybox.xml。
  3. 設定 target 含 com.google.android.gms 與你的目標 App。
# 確認 keybox 真的放進去了（大小 > 0 才算）
adb shell su -c "ls -l /data/adb/tricky_store/keybox.xml"

keybox 是這條路線唯一的真瓶頸

它從 OEM 外洩、會被 Google 持續撤銷，今天能過明天可能就失效。A13 的 DEVICE 因此不保證長期穩定。若你的 App 其實只要 BASIC，回路線 A 反而省心。

驗證

  • Play Integrity API Checker（henrikherzig）Play Store — 一眼看 BASIC / DEVICE / STRONG 各自 PASS / FAIL。
  • Play 商店內建：設定 → 連點「Play 商店版本」7 次 → 一般 → 開發者選項 → 驗證完整性。

檢查模組與日誌：

adb shell su -c "ls -la /data/adb/modules/"
adb shell su -c "magisk --denylist ls"
adb shell su -c "logcat | grep -E '(PIF|TrickyStore)'"

故障排除

  • BASIC 都不過：多半是 Shamiko 沒接手，確認「強制執行拒絕清單」是關，且 DenyList 有勾 GMS。
  • BASIC 過、DEVICE 不過（A10）：重生 PIF 指紋（上面指令），清 GMS 快取再測。
  • DEVICE 不過（A13）：你的 keybox 可能已被撤銷，換一把；或確認 TrickyStore target 有含 GMS。
  • 過幾天又失效：Google 更新偵測或撤銷 keybox 是常態，更新 PIF 模組 / 換 keybox。
